## Résumé des Décisions V2

### Hardware V2

- **Boîtier** : bloc de bois massif 30×30×5 cm, pads aluminium 4mm incrustés, arrangement circulaire/radial.
- **3 boutons** : gauche (bank select), droit (scale/arp controls), arrière (battery/setup). Tous sur de nouveaux GPIOs à définir. Remplacent les GPIO2/GPIO3 de la V1.
- **3 potentiomètres** : gauche (feel/son), droit (notes/rythme), arrière (config/tempo). Le pot V1 (GPIO1) est remplacé par 3 nouveaux ADC à définir.
- **Nature des pads capacitifs** : surface de contact humide, pas force mécanique. Aftertouch = oui. Velocity liée à la pression = non fiable. Velocity fixe pour le moment.

### Architecture Globale

- **Fresh start** — pas de backward compatibility avec V1.
- **8 banks toujours vivantes** — le bank select (bouton gauche) change juste quelle bank est au "premier plan" (reçoit les pads). Les ArpEngines en mode HOLD continuent en arrière-plan.
- **Deux types de bank** : NORMAL (pads → notes + poly-aftertouch) et ARPEG (arpégiateur, pas d'aftertouch, velocity fixe). Max 4 ARPEG simultanées.
- **Calibration** : identique V1 (Tool 1 Pressure Calibration ne change pas).

### Les 3 Couches de Mapping

1. **Pad Ordering** (physique → logique) : `padOrder[48]`, rang 0-47 du grave à l'aigu. Fait une fois, valable pour toutes les banks. Tool 2.
2. **Scale Config** (par bank, runtime) : chromatique ou gamme (root + mode). Root = base note partout (en chromatique, root=D → commence à D2 ; en gamme, root=D → D Dorian/etc). Changeable à chaud via bouton droit. Persisté NVS par bank.
3. **Control Pad Assignment** : 31 pads assignés à des fonctions de contrôle. Un pad = un seul rôle. Tool 3.

### Boutons

- **Bouton gauche (hold + pad)** : bank select. 8 pads. Même logique que V1 (arm/disarm debounce). Au switch : all notes off pour NORMAL, rien pour ARPEG (l'arpège vit/meurt par sa propre logique).
- **Bouton droit (hold + pad)** : scale + arp controls. 15 pads communs (7 root A-G + 7 mode Ionian→Locrian + 1 chromatique toggle). 7 pads arp supplémentaires sur banks ARPEG (5 patterns + 1 octave cycle + 1 HOLD toggle). Aucune note MIDI émise pendant le hold.
- **Bouton arrière** : battery gauge + setup mode entry (hold 3s au boot).

### Arpégiateur

**Deux sous-modes déterminés par HOLD :**

**HOLD OFF (mode live)** :
- Press pad = note ajoutée. Release = note retirée. Tous doigts levés = arpège s'arrête naturellement (liste vide).
- Le pad play/stop joue une note normale (pas de rôle spécial).
- Le double-tap n'a pas de sens (lever le doigt suffit).
- Le BankManager ne gère aucune logique d'arrêt — l'arpège meurt parce que l'utilisateur lève les doigts (pour switcher de bank), pas à cause du bank switch lui-même.

**HOLD ON (mode persistant)** :
- Press pad = note ajoutée (persiste après relâche). Double-tap (< 300ms) = retire la note.
- Pad play/stop = toggle transport. Stop puis Play repart du début.
- Bank switch = arpège continue en background.
- Retour sur la bank = on peut ajouter/retirer des notes.

**Pad Play/Stop** :
- Assigné dans Tool Pad Roles (section Arp). Même pad physique pour toutes les banks ARPEG.
- Comportement contextuel : sur NORMAL → note normale. Sur ARPEG HOLD OFF → note normale. Sur ARPEG HOLD ON → play/stop toggle.
- Seul pad de contrôle actif en mode jeu (les 30 autres ne sont actifs que pendant les holds).

**Specs arpège** :
- 5 patterns : Up, Down, Up-Down, Random, Order (ordre de press).
- Pile : jusqu'à 48 notes. Avec 4 octaves = 192 steps max.
- Octave range 1-4 (cycle via pad).
- Division rythmique : pot droit (ronde → 1/32).
- Gate length : pot gauche.
- Swing : btn droit + pot gauche.
- Base velocity : btn droit + pot droit (non-live).
- Velocity variation : btn gauche + pot droit (live, ±% random).

### 3 Potentiomètres

Fonctions contextuelles selon le type de bank (NORMAL vs ARPEG). Système catch sur tous les pots (le pot doit passer par la valeur stockée avant de prendre le contrôle).

**Pot Gauche (feel/son)** :
- NORMAL pot seul = response shape. Btn droit + pot G = slew rate.
- ARPEG pot seul = gate length. Btn droit + pot G = swing.
- Btn gauche + pot G = LIBRE (les deux modes).

**Pot Droit (notes/rythme)** :
- NORMAL pot seul = LIBRE. Btn gauche + pot D = AT deadzone. Btn droit + pot D = LIBRE.
- ARPEG pot seul = division rythmique. Btn gauche + pot D = velocity variation. Btn droit + pot D = base velocity.

**Pot Arrière (config)** :
- Pot seul = tempo interne (40-240 BPM). Btn gauche + pot arr = pad sensitivity. Btn droit + pot arr = LIBRE.

**Stockage des valeurs pot** :
- Paramètres NORMAL (response shape, slew rate, AT deadzone) = **GLOBAUX** (un seul set partagé par toutes les banks NORMAL).
- Paramètres ARPEG (gate, swing, division, base velocity, velocity variation) = **PAR BANK** (chaque bank ARPEG a ses propres valeurs).
- Quand on switch de bank ARPEG, le catch se réinitialise (pot physique doit re-catcher la valeur de la nouvelle bank).

### MIDI Clock / Tempo

- Réception MIDI Clock (0xF8, 0xFA, 0xFC) via USB et BLE.
- Priorité : USB clock > BLE clock > dernier tempo reçu > tempo interne (pot arrière).
- **PLL logicielle** : lisse le jitter BLE (±15ms → ±1-2ms). Calcule le tempo moyen sur 24 ticks. Génère un clock interne régulier. Les ArpEngines se synchronisent au clock lissé.

### Optimisations Architecture

- **Double buffer** remplace le mutex V1. Core 0 écrit dans buffer A, Core 1 lit buffer B, swap atomique. Core 1 ne bloque jamais.
- **NVS task dédiée** (FreeRTOS, priorité basse). Le loop principal ne bloque jamais sur flash write.
- **BLE connection interval** configurable dans Settings : Low Latency (7.5ms), Normal (15ms), Battery Saver (30ms).
- **Séparation chemin critique** dans le loop : MIDI d'abord (notes, arp ticks, flush), secondaire ensuite (pots, LEDs, NVS, WiFi).

### PotRouter

Module unique qui lit les 3 ADC, résout les combos boutons, applique le catch, expose des getters. Table de bindings déclarative — ajouter un paramètre = ajouter une ligne. Les consumers (ArpEngine, CapacitiveKeyboard, etc.) appellent des getters, ne savent pas quel pot physique.

### Setup Mode V2

```
[1] Pressure Calibration    — inchangé V1
[2] Pad Ordering            — "touche du grave à l'aigu", positions 1-48, pas de base note
[3] Pad Roles               — unifié (bank + scale + arp), sous-menu, grille couleur, collision check
[4] Bank Config             — NORMAL/ARPEG par bank (max 4 ARPEG)
[5] Settings                — profile, sensitivity, AT rate, deadzone, BLE interval
[0] Reboot & Exit
```

**Tool 3 Pad Roles** : un seul tool avec sous-menu [1] Bank Pads (8), [2] Scale Pads (15), [3] Arp Pads (8), [4] View All / Collisions. Grille 4×12 avec couleurs (bleu=bank, vert=scale, jaune=arp, rouge=collision). Refuse de sauvegarder avec collisions. 31 pads total, 17 libres.

### WiFi Status Broadcast (P3, doc séparé)

- WiFi AP on-demand (désactivé par défaut, activé par combo bouton au boot).
- Page web embarquée en flash (~50KB).
- WebSocket pour le stream JSON.
- Données : bank state, scale config, arp state, BPM, batterie, connexions, pot actif (nom + valeur). Pas de données de pads.
- ~50-100 bytes/sec en pratique. Impact négligeable.

### Performance

- CPU Core 0 : ~92% (sensing, inchangé). CPU Core 1 : ~16%.
- BLE MIDI : 30-50% en worst case (4 arps + 16 pads AT). Goulot potentiel.
- SRAM : ~5%. Flash : ~6%.
- Mitigation BLE : noteOn/Off bypass queue (prioritaires), aftertouch en queue overflow toléré.

### Plan d'Implémentation (5 phases)

1. **Infrastructure** : arborescence, double buffer, NVS task, BankSlot NORMAL, managers, main.cpp.
2. **Pots + Scale** : PotRouter, ToolPadOrdering, ScaleResolver, ScaleManager, root=base note.
3. **Pad Roles + Bank Config** : ToolPadRoles, ToolBankConfig, ToolSettings.
4. **Arpégiateur** : ClockManager+PLL, ArpEngine, ArpScheduler, play/stop, double-tap, gate/swing/velocity.
5. **Polish** : LEDs, edge cases, factory reset, WiFi status (P3).

### Conventions

- `s_` static globals, `_` members, `SCREAMING_SNAKE_CASE` constantes.
- `#if !PRODUCTION_MODE` pour debug.
- Pas de `new`/`delete` runtime. Instanciation statique.
- C++17, Arduino, PlatformIO `esp32-s3-devkitc-1`.

### Ne PAS Modifier

- `platformio.ini` (sauf lib_deps si nécessaire).
- Constantes de tuning pression dans `HardwareConfig.h`.
- La pressure pipeline dans `CapacitiveKeyboard.cpp`.
